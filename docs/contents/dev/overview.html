Overview

SSSG is a super-lightweight framework for composing static website with live preview.

As SSSG is just a compilation of many useful npm modules, this how-to-document shows you little about what SSSG does but much about how SSSG utilizes these npm modules.

The goal for this framework are:

  • to code web service with developer friendly transpile languages like Pug, Stylus, ES6 Javascript.
  • to see real time preview of a web page on coding. (WYSIWYG)
  • to export resulted static web files anywhere.

I originally wanted prototyping tool for website and found some wireframing services. Those services is useful in that users don't need to code anything to outline site design.
However, as a programmer myself,  I NEEDED CODE to spec out web designs.
By the reason above, this framework is targeting coders who want to share prototype design based on text.

Simple = The rest is yours

SSSG does only a few things.

Basically, it

  • compiles `index.pug` in source folder into `index.html` in destination folder.
  • compiles `main.styl` in source folder into `main.css` in destination folder.
  • compiles `main.js` in source folder into `main.js` in destination folder.
  • copies image files and other misc files into destination folder.

This is so simple, isn't it? Actually SSSG is not a framework but just a convention.

While SSSG does just simple works, you can fully take advantages of pug/stylus/es6 features without concerns.

Pug

Pug is equipped with code template system. If your website utilizes the same layout over several pages, you should split the pages into static components(sidebar, header, footer, etc) and dynamic components.
Pug encourages you to do so.
How you structure your website components is up to you. The choice is yours.

Stylus

Haven't you hear DRY coding? If no, you've got a chance to learn it. As well as Pug, Stylus makes you code less, by representing parent-child structure as an indent. To fully speed up coding, you should try IDE that supports Stylus.
Additionally, Stylus also supports modular system. It can `import` any other stylus files. You can construct your own modular structure as clean as you feel comfortable.
Of course, you can write all style information only in `main.styl`.

ES6

Unlike Pug/Stylus, ES6 itself is not equipped with less-code syntax. But it has asynchronous system with `Promise`, native class support, and like Stylus, `import` statement.
With `class` and `import` feature, you can fully modularize your front-end business logic into reusable and structured framework.

Try it out

Requirements

You need to have Git and NodeJS 6.x installed your computer.
Check out official installation manual.

After you installed these requirements, install SSSG globally on your computer.

$ npm install -g git+https://github.com/Hinaser/sssg

Let's try

$ cd [work folder]
$ sssg try

After dispatching the above command, browser will launch and open ./docs/index.html created in your working folder.
